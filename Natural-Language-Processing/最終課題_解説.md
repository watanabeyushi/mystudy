## 全体の流れ

このコードは、**JALのTripAdvisorレビュー**に対して、**LSTM（Long Short-Term Memory）**という深層学習アルゴリズムを使用して感情分析を行うプログラムです。

### 処理の流れ（概要図）

```
1. データ読み込み
   ↓
2. 形態素解析（日本語を単語に分割）
   ↓
3. テキスト前処理（単語を数値に変換）
   ↓
4. LSTMモデルの構築
   ↓
5. モデルの訓練
   ↓
6. 予測と評価
   ↓
7. 可視化とサマリー
```

### 何をしているのか？

1. **レビューテキストを読む** → CSVファイルから日本語レビューを読み込む
2. **日本語を単語に分割する** → 形態素解析で「素晴らしいサービス」→「素晴らしい」「サービス」に分割
3. **単語を数値に変換する** → コンピュータが理解できるように数字に変換
4. **AIモデルを作る** → LSTMという深層学習モデルを構築
5. **モデルを訓練する** → レビューの評価（rating）を参考に、感情を学習させる
6. **感情を予測する** → 訓練したモデルで、新しいレビューの感情を予測
7. **結果を表示する** → グラフや統計で結果を可視化

## セル1: 形態素解析

### コードの説明

```python
from janome.tokenizer import Tokenizer
tokenizer = Tokenizer()

# 抽出する品詞を指定
ALLOWED_POS = [
    '名詞',      # 名詞全般（サービス、食事、座席など）
    '形容詞',    # 形容詞（良い、悪い、素晴らしいなど）
    '動詞',      # 動詞（満足する、感謝するなど）
    '副詞',      # 副詞（とても、非常になど）
]

def tokenize_japanese(text, allowed_pos=ALLOWED_POS):
    tokens = tokenizer.tokenize(text)
    filtered_tokens = []
    for token in tokens:
        # 品詞情報を取得
        pos = token.part_of_speech.split(',')[0]
        # 指定した品詞のみを抽出
        if pos in allowed_pos:
            surface = token.surface.strip()
            if surface and len(surface) > 1:
                filtered_tokens.append(surface)
    return filtered_tokens
```

### 何をしているか？

**形態素解析**とは、日本語の文章を単語単位に分割する処理です。さらに、**品詞制限**により、感情分析に有用な品詞のみを抽出します。

#### 具体例

```
入力: "素晴らしいサービスでした"

形態素解析後（全品詞）:
  ["素晴らしい", "サービス", "でした"]

品詞制限後（名詞、形容詞、動詞、副詞のみ）:
  ["素晴らしい", "サービス"]
  ※ "でした"（助動詞）は除外される
```

### 品詞制限のメリット

1. **ノイズの削減**: 助詞や助動詞など、感情分析にあまり役立たない単語を除外
2. **精度の向上**: 意味のある単語のみを使用することで、モデルの学習が効率的に
3. **データ量の削減**: 不要な単語を除外することで、処理速度が向上

### 抽出される品詞の説明

- **名詞**: 物事の名前（サービス、食事、座席、スタッフなど）
- **形容詞**: 状態や性質を表す（良い、悪い、素晴らしい、不快など）
- **動詞**: 動作や状態を表す（満足する、感謝する、困るなど）
- **副詞**: 程度や様子を表す（とても、非常に、少しなど）

### 除外される品詞の例

- **助詞**: は、が、を、に、でなど（文法的な役割のみ）
- **助動詞**: です、ます、でしたなど（丁寧語の表現）
- **接続詞**: しかし、そしてなど（文の接続のみ）

### なぜ必要か？

- 日本語は単語の境界が明確でない（英語のようにスペースで区切られていない）
- コンピュータが文章を理解するには、まず単語に分割する必要がある
- 例：「今日は良い天気」→「今日」「は」「良い」「天気」に分割

### janomeとは？

- **janome**: 日本語の形態素解析ライブラリ（Python用）
- 辞書ベースで、日本語の単語を正確に分割できる
- MeCabの代替として、インストールが簡単

### 処理の流れ

1. `tokenizer.tokenize(text)`: 文章を形態素（単語）に分割
2. `token.part_of_speech`: 各形態素の品詞情報を取得
3. 品詞フィルタリング: 指定した品詞（名詞、形容詞、動詞、副詞）のみを抽出
4. `token.surface`: 各形態素の表層形（実際の文字列）を取得
5. フィルタリング: 1文字以下の単語を除外

### 初学者への補足

- **形態素**: 意味を持つ最小の言語単位（単語や助詞など）
- **トークン**: 分割された単語のことを指す
- **品詞**: 単語の文法的な分類（名詞、動詞、形容詞など）
- この処理により、「文章」→「意味のある単語のリスト」に変換される
- 品詞制限により、感情分析に有用な単語のみが抽出される

```python
max_length = int(np.percentile(seq_lengths, 95))  # 95パーセンタイル
```

**何をしているか？**

**具体例：**
```
元の長さが異なる:
  レビュー1: [1, 2, 3]           (長さ3)
  レビュー2: [4, 5, 6, 7, 8]     (長さ5)

パディング後（max_length=5）:
  レビュー1: [1, 2, 3, 0, 0]     (後ろに0を追加)
  レビュー2: [4, 5, 6, 7, 8]     (そのまま)
```

**なぜ95パーセンタイル？**
- 極端に長いレビュー（外れ値）の影響を避けるため
- データの95%がこの長さ以下になるように設定

## セル3: LSTMモデルの構築

### コードの説明

```python
model = Sequential([
    Embedding(input_dim=vocab_size, output_dim=embedding_dim, input_length=max_length),
    LSTM(lstm_units, dropout=0.2, recurrent_dropout=0.2),
    Dense(32, activation='relu'),
    Dropout(0.3),
    Dense(3, activation='softmax')
])
```

### モデルの構造（層ごとの説明）

#### 1. Embedding層

```python
Embedding(input_dim=vocab_size, output_dim=embedding_dim, input_length=max_length)
```

**何をしているか？**
- 単語のID番号を、意味を表すベクトル（数値のリスト）に変換
- 例: 単語ID「1」→ [0.2, -0.5, 0.8, ...]（128次元のベクトル）

**なぜ必要か？**
- 単語の意味的な関係を数値で表現できる
- 似た意味の単語は、似たベクトルになる

**具体例：**
```
"良い" → [0.1, 0.2, 0.3, ...]
"素晴らしい" → [0.12, 0.18, 0.31, ...]  （似たベクトル）
"悪い" → [-0.1, -0.2, -0.3, ...]  （反対のベクトル）
```

#### 2. LSTM層

```python
LSTM(lstm_units, dropout=0.2, recurrent_dropout=0.2)
```

**LSTMとは？**
- **Long Short-Term Memory**（長短期記憶）の略
- 時系列データ（単語の並び）を処理する深層学習のアルゴリズム
- 文脈を理解できる（前後の単語の関係を考慮）

**何をしているか？**
- 単語の順序を考慮して、文章全体の意味を理解
- 例：「良い」の後に「サービス」が来ると、ポジティブな意味になる

**LSTMの特徴：**
- **記憶機能**: 前の単語の情報を保持できる
- **長期依存**: 文章の最初と最後の関係も捉えられる
- **64ユニット**: 64個のニューロンで処理

**Dropoutとは？**
- 過学習を防ぐための正則化手法
- 訓練時に一部のニューロンをランダムに無効化
- `dropout=0.2`: 20%のニューロンを無効化

#### 3. Dense層（全結合層）

```python
Dense(32, activation='relu')
Dense(3, activation='softmax')
```

**何をしているか？**
- 第1層: 32個のニューロンで特徴を変換
- 第2層: 3クラス（negative, positive, neutral）に分類

**活性化関数：**
- `relu`: 負の値を0にする関数（非線形変換）
- `softmax`: 3つのクラスの確率を計算（合計が1になる）

**具体例（softmaxの出力）：**
```
[0.1, 0.8, 0.1]  → positive（80%の確率）
[0.7, 0.2, 0.1]  → negative（70%の確率）
```

### モデルの全体像

```
入力: [1, 2, 3, 0, 0]  (単語IDのリスト)
  ↓
Embedding層: 各IDを128次元ベクトルに変換
  ↓
LSTM層: 文脈を考慮して特徴を抽出
  ↓
Dense層: 特徴を変換
  ↓
出力: [0.1, 0.8, 0.1]  (negative, positive, neutralの確率)
```

### 初学者への補足

- **ニューロン**: 脳の神経細胞を模した計算単位
- **層**: ニューロンの集まり
- **深層学習**: 複数の層を重ねた機械学習モデル
- **過学習**: 訓練データにだけ適合して、新しいデータに弱くなる現象

## セル4: モデルの訓練

### コードの説明

```python
X_train, X_test, y_train, y_test = train_test_split(
    X, y_categorical, test_size=0.2, random_state=42, stratify=y
)

history = model.fit(
    X_train, y_train,
    batch_size=32,
    epochs=10,
    validation_data=(X_test, y_test)
)
```

### 何をしているか？

#### 1. データの分割

```python
train_test_split(X, y_categorical, test_size=0.2, stratify=y)
```

**何をしているか？**
- データを訓練用（80%）と検証用（20%）に分割
- `stratify=y`: 各クラスの比率を保つ（バランスを保つ）

**なぜ分割するか？**
- 訓練データ: モデルを学習させる
- 検証データ: モデルの性能を評価する（未知のデータでの性能）

#### 2. モデルの訓練（早期停止付き）

```python
# 早期停止の設定
early_stopping = EarlyStopping(
    monitor='val_loss',        # 監視する指標（検証損失）
    patience=3,                # 改善が見られないエポック数を何回まで許容するか
    restore_best_weights=True, # 最良の重みを復元
    verbose=1                  # 停止時にメッセージを表示
)

# モデルの訓練
model.fit(
    X_train, y_train,
    batch_size=32,
    epochs=50,                 # 最大エポック数（早期停止で自動的に停止）
    validation_data=(X_test, y_test),
    callbacks=[early_stopping] # 早期停止コールバックを追加
)
```

**パラメータの説明：**
- `batch_size=32`: 32件ずつまとめて処理（効率的）
- `epochs=50`: 最大エポック数（早期停止により実際は少なくなる可能性がある）
- `validation_data`: 各エポック後に検証データで性能を確認
- `callbacks=[early_stopping]`: 早期停止コールバックを追加

**早期停止（Early Stopping）とは？**

過学習を防ぐための手法で、検証データの性能が改善しなくなったら自動的に訓練を停止します。

**早期停止のパラメータ：**
- `monitor='val_loss'`: 検証損失を監視（値が小さくなるほど良い）
- `patience=3`: 3エポック連続で改善が見られなければ停止
- `restore_best_weights=True`: 最良の重み（検証損失が最小だった時点）を復元

**早期停止の動作：**
1. 各エポック後に検証損失を確認
2. 検証損失が改善した場合：訓練を継続
3. 検証損失が改善しない場合：patienceカウンタを増やす
4. patience回連続で改善しない場合：訓練を停止し、最良の重みを復元

**訓練の流れ：**
1. 訓練データを32件ずつ取り出す
2. モデルに予測させる
3. 正解と比較して誤差を計算
4. 誤差を最小化するようにモデルのパラメータを更新
5. 全データを処理するまで繰り返し
6. 検証データで性能を確認
7. 早期停止の条件をチェック
8. 条件を満たすまで、または最大エポック数に達するまで繰り返し

### 初学者への補足

- **エポック**: 全データを1回処理すること
- **バッチ**: まとめて処理するデータの塊
- **検証データ**: 訓練に使わない、性能評価用のデータ
- **学習**: モデルがデータからパターンを学ぶこと
- **早期停止**: 過学習を防ぐための手法。検証データの性能が改善しなくなったら訓練を自動停止
- **patience**: 早期停止のパラメータ。何エポック改善が見られなければ停止するかを指定

## セル5: 予測と評価

### コードの説明

```python
y_pred_proba = model.predict(X)
y_pred = np.argmax(y_pred_proba, axis=1)
```

### 何をしているか？

#### 1. 予測の実行

```python
y_pred_proba = model.predict(X)
```

**何をしているか？**
- 訓練したモデルで、全データの感情を予測
- 出力は確率のリスト（例: [0.1, 0.8, 0.1]）

#### 2. ラベルの決定

```python
y_pred = np.argmax(y_pred_proba, axis=1)
```

**何をしているか？**
- 確率が最も高いクラスを選択
- 例: [0.1, 0.8, 0.1] → 1（positive）

#### 3. 精度評価

```python
accuracy = accuracy_score(y, y_pred)
classification_report(y, y_pred)
confusion_matrix(y, y_pred)
```

**評価指標：**
- **精度（Accuracy）**: 全体の正解率
- **分類レポート**: 各クラスの精度、再現率、F1スコア
- **混同行列**: 予測と正解の対応表

**混同行列の例：**
```
        予測
       neg pos neu
正 neg  50  5   5
解 pos  3   80  2
   neu  2   3   54
```

### 初学者への補足

- **精度**: 正しく予測できた割合
- **再現率**: 実際のpositiveのうち、どれだけ予測できたか
- **F1スコア**: 精度と再現率のバランス

## アルゴリズムの詳細解説

### 1. 形態素解析アルゴリズム

#### 仕組み

1. **辞書マッチング**: 事前に用意した辞書と照合
2. **品詞推定**: 単語の品詞（名詞、動詞など）を判定
3. **分割**: 文章を形態素単位に分割

#### janomeの特徴

- **辞書ベース**: 辞書に登録された単語を認識
- **未知語処理**: 辞書にない単語も推定
- **軽量**: MeCabより軽量でインストールが簡単

### 2. LSTMアルゴリズム

#### LSTMの構造

LSTMは以下の3つのゲートで構成されています：

1. **忘却ゲート（Forget Gate）**
   - 古い情報を忘れるかどうかを決定
   - 不要な情報を削除

2. **入力ゲート（Input Gate）**
   - 新しい情報を記憶するかどうかを決定
   - 重要な情報を保存

3. **出力ゲート（Output Gate）**
   - どの情報を出力するかを決定
   - 必要な情報だけを出力

#### LSTMの数式（簡略版）

```
f_t = σ(W_f · [h_{t-1}, x_t] + b_f)  # 忘却ゲート
i_t = σ(W_i · [h_{t-1}, x_t] + b_i)  # 入力ゲート
C_t = f_t * C_{t-1} + i_t * tanh(W_C · [h_{t-1}, x_t] + b_C)  # セル状態
o_t = σ(W_o · [h_{t-1}, x_t] + b_o)  # 出力ゲート
h_t = o_t * tanh(C_t)  # 隠れ状態
```

**記号の説明：**
- `σ`: シグモイド関数（0-1の値を出力）
- `W`: 重み行列（学習されるパラメータ）
- `b`: バイアス（学習されるパラメータ）
- `h_t`: 時刻tの隠れ状態
- `C_t`: 時刻tのセル状態（長期記憶）

#### なぜLSTMが感情分析に適しているか？

1. **文脈の理解**: 前後の単語の関係を考慮
   - 例：「良い」の後に「サービス」→ ポジティブ
   - 例：「良い」の後に「とは言えない」→ ネガティブ

2. **長期依存**: 文章の最初と最後の関係も捉えられる
   - 例：「最初は良かったが、最後は最悪だった」

3. **系列データの処理**: 単語の順序を考慮できる

### 3. 深層学習の訓練アルゴリズム

#### バックプロパゲーション（誤差逆伝播法）

1. **順伝播**: 入力から出力まで計算
2. **誤差計算**: 予測と正解の差を計算
3. **逆伝播**: 誤差を後ろから前に伝播
4. **パラメータ更新**: 誤差を減らすように重みを更新

#### 最適化アルゴリズム（Adam）

- **Adam**: 適応的モーメント推定
- 学習率を自動調整
- 効率的に最適解に収束

### 4. 評価指標の意味

#### 精度（Accuracy）

```
精度 = 正解数 / 全体数
```

- 全体のうち、どれだけ正しく予測できたか

#### 再現率（Recall）

```
再現率 = 正しく予測したpositive / 実際のpositive
```

- 実際のpositiveのうち、どれだけ見つけられたか

#### 適合率（Precision）

```
適合率 = 正しく予測したpositive / 予測したpositive
```

- 予測したpositiveのうち、どれだけ正しかったか

#### F1スコア

```
F1 = 2 × (適合率 × 再現率) / (適合率 + 再現率)
```

- 適合率と再現率のバランス

## 考察

### 分析結果の概要

実際の分析結果から、以下のような特徴が観察されました：

- **総レビュー数**: 5,446件
- **平均評価**: 4.24（5段階評価）
- **モデル精度**: 0.890（89.0%）
- **平均予測確信度**: 0.902（90.2%）

### 1. データの偏りに関する考察

#### 評価分布の特徴

分析結果から、評価分布に大きな偏りが見られます：

- **5つ星**: 2,552件（46.9%）- 最多
- **4つ星**: 2,096件（38.5%）
- **3つ星**: 489件（9.0%）
- **2つ星**: 152件（2.8%）
- **1つ星**: 157件（2.9%）

**考察：**
- ポジティブなレビュー（4-5つ星）が全体の85.4%を占める
- ネガティブなレビュー（1-2つ星）はわずか5.7%と非常に少ない
- この偏りは、TripAdvisorの特性（満足した顧客がレビューを書く傾向）を反映している

**影響：**
- モデルがポジティブなレビューに偏って学習する可能性
- ネガティブなレビューの予測精度が低くなる可能性

### 2. 感情分析結果の考察

#### LSTM予測結果の特徴

- **positive**: 5,062件（92.9%）
- **negative**: 382件（7.0%）
- **neutral**: 2件（0.0%）

**考察：**
- 予測結果も評価分布と同様に、positiveが圧倒的に多い
- neutralがほぼ0件ということは、モデルが「positive」か「negative」の二択に近い判断をしている
- これは、データの偏りがモデルの予測に影響を与えている可能性を示唆

**問題点：**
- 実際には「neutral」なレビュー（3つ星）が489件あるが、モデルはほとんど「positive」と予測
- モデルが中間的な感情を捉えられていない可能性

### 3. モデル性能の考察

#### 精度と一致率

- **全体精度**: 0.890（89.0%）
- **ratingとの一致率**: 0.890（89.0%）

**考察：**
- 89%の精度は、感情分析タスクとしては良好な結果
- ただし、データの偏りを考慮すると、単純に「positive」と予測するだけでも高い精度が出る可能性がある

**注意点：**
- 精度だけでは不十分。各クラス（positive/negative/neutral）の精度を個別に確認する必要がある
- 特に、少数派のnegativeクラスの精度が重要

### 4. 訓練履歴からの考察

#### 精度の推移

- **訓練精度**: 約0.869（最終エポック）
- **検証精度**: 約0.857（最終エポック）

**考察：**
- 訓練精度と検証精度の差は約0.012と小さいが、エポック7-8で検証精度が低下
- これは軽度の過学習の兆候を示している可能性がある
- **早期停止の効果**: 早期停止を実装することで、検証損失が改善しなくなった時点で自動的に訓練を停止し、過学習を防ぐことができる

#### 損失の推移

- **訓練損失**: 約0.395（最終エポック）
- **検証損失**: 約0.455（最終エポック）

**考察：**
- 訓練損失は順調に減少しているが、検証損失の減少は緩やか
- エポック7以降で訓練損失が急激に減少しているが、検証損失はそれほど改善していない
- これは、モデルが訓練データに過度に適合している可能性を示唆

### 5. 品詞制限の効果

品詞制限（名詞、形容詞、動詞、副詞のみ）により、以下の効果が期待されます：

**メリット：**
- ノイズの削減：助詞や助動詞など、感情分析に不要な単語を除外
- 処理速度の向上：データ量が減り、処理が高速化
- モデルの学習効率：意味のある単語のみで学習することで、モデルが重要な特徴を捉えやすくなる

**注意点：**
- 一部の助詞や助動詞が感情表現に重要な役割を果たす場合もある（例：「とても良い」の「とても」）
- ただし、副詞として「とても」は抽出されるため、この問題は軽減されている

### 6. 改善点と今後の課題

#### データの不均衡への対応

1. **クラス重み付け（Class Weighting）**
   - 少数派クラス（negative）に高い重みを付けて学習
   - モデルが少数派クラスを重視するように調整

2. **データ拡張（Data Augmentation）**
   - ネガティブなレビューを人工的に増やす
   - ただし、自然言語では難しい

3. **アンダーサンプリング/オーバーサンプリング**
   - ポジティブなレビューを減らす、またはネガティブなレビューを増やす
   - ただし、データ量が減る可能性がある

#### モデルの改善

1. **ハイパーパラメータの調整**
   - エポック数を増やす（早期停止により過学習を防げる）
   - LSTMユニット数を調整
   - ドロップアウト率の調整
   - 早期停止のpatience値を調整（デフォルトは3）

2. **モデル構造の改善**
   - Bidirectional LSTMの使用（双方向から文脈を理解）
   - Attention機構の追加（重要な単語に注目）
   - より深いネットワークの構築

3. **事前学習済みモデルの活用**
   - BERTやTransformerなどの事前学習済みモデルを使用
   - より高精度な感情分析が可能

#### 評価指標の改善

1. **クラス別の評価**
   - 各クラス（positive/negative/neutral）の精度、再現率、F1スコアを個別に確認
   - 特に少数派クラスの性能を重視

2. **混同行列の詳細分析**
   - どのクラスをどのクラスと間違えやすいかを分析
   - 誤分類のパターンを理解

### 7. 実用的な観点からの考察

#### ビジネスへの応用

1. **顧客満足度の把握**
   - ポジティブなレビューが多いことは、サービス品質が高いことを示唆
   - ただし、ネガティブなレビューも重要な改善点を示している

2. **課題の特定**
   - ネガティブなレビューから、具体的な問題点を抽出
   - 改善すべき点を優先順位付け

3. **リアルタイム分析**
   - 新しいレビューに対して自動的に感情分析を実行
   - 問題の早期発見と対応

#### 限界と注意点

1. **文脈の理解**
   - 現在のモデルは文脈をある程度理解しているが、複雑な表現（皮肉、二重否定など）には対応できない可能性
   - 例：「悪くない」は実際にはポジティブだが、単語レベルでは「悪い」が含まれる

2. **文化的な違い**
   - 日本語特有の表現（遠回しな表現、謙遜など）への対応
   - 例：「まあまあ」は実際には中程度の評価だが、文脈によってはネガティブにも解釈される

3. **ドメイン依存性**
   - 航空会社のレビューで訓練したモデルは、他のドメイン（レストラン、ホテルなど）では精度が低下する可能性

### 8. 総合的な評価

#### 成功している点

- **高い精度**: 89%の精度は実用的なレベル
- **効率的な処理**: 品詞制限により、処理が効率化
- **実装の簡潔性**: シンプルなLSTMモデルで良好な結果

#### 改善が必要な点

- **データの不均衡**: ネガティブなレビューの予測精度を向上させる必要
- **過学習の抑制**: 訓練データへの過度な適合を防ぐ
- **neutralクラスの予測**: 中間的な感情を適切に捉える

#### 今後の展望

1. **より高度なモデル**: BERTやTransformerなどの事前学習済みモデルの活用
2. **マルチタスク学習**: 感情分析と同時に、トピック分類なども行う
3. **説明可能性**: なぜその感情と判断したかを説明できるモデルの構築
